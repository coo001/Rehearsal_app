# AI 리허설 MVP 정의서

> 배우가 AI 상대역과 자연스럽게 대본을 연습할 수 있는 최소 기능 제품

---

## 1. 한 줄 요약

**고정 대본(~10줄) 기반으로, AI가 상대역 대사를 TTS로 읽어주고 사용자가 말을 끝내면 자동으로 다음 턴이 넘어가는 리허설 웹앱**

---

## 2. 핵심 사용자 시나리오

```
1. 앱 접속
2. 자신이 맡을 역할 선택 (캐릭터 A 또는 B)
3. [리허설 시작] 버튼 클릭
4. AI 대사 → TTS 자동 재생
5. 내 대사 차례 → 마이크 자동 활성화 (listening 상태 표시)
6. 내가 말을 끝내면 → 자동으로 다음 AI 대사로 넘어감
7. 대본 끝날 때까지 반복
8. 완료 화면 표시
```

## 4. 기능 범위 (In / Out)

### ✅ MVP에 포함

| 기능 | 설명 |
|------|------|
| 역할 선택 | 사용자가 캐릭터 A 또는 B 중 하나를 선택 |
| AI 대사 TTS 재생 | 상대역 대사를 TTS 음성으로 자동 재생 |
| 사용자 턴 표시 | 내 대사 차례임을 시각적으로 명확히 표시 |
| VAD 자동 트리거 | 사용자가 말을 마치면 자동으로 다음 턴 진행 |
| 수동 진행 버튼 | VAD 미작동 시 버튼으로 수동으로 다음 턴 넘기기 |
| 전체 대본 진행 표시 | 현재 몇 번째 대사인지 표시 |
| 완료 화면 | 대본이 끝나면 완료 메시지 + 다시 시작 버튼 |

### ❌ MVP에서 제외

- 대본 입력 / 분석 (하드코딩 대본 사용)
- PDF 업로드
- 목소리 선택 UI (기본 목소리 고정)
- 연습 범위 선택
- STT (음성 인식 / 받아쓰기) — 트리거만 있으면 됨
- 감정 분석, beat/subtext 분석
- 세션 저장 / 복원
- 모바일 최적화

## 5. 핵심 기술 결정

### 5-1. TTS 제공자

| 옵션 | 추천 이유 |
|------|----------|
| **OpenAI TTS** (`tts-1`) | API 키 하나, 간단한 HTTP 호출, 한국어 품질 양호 |
| ElevenLabs | 더 자연스럽지만 설정 복잡 |

**MVP 추천: OpenAI TTS**  
음성은 `alloy` 또는 `nova` 고정. 캐릭터별로 다른 목소리를 쓰면 더 좋음 (예: A=`alloy`, B=`nova`).

### 5-2. 사용자 말 끝 감지 (VAD)

브라우저에서 사용자가 말을 끝냈는지 감지하는 방법. 아래 두 가지 중 하나를 선택.

**옵션 A — Web Speech API (가장 단순)**

```javascript
const recognition = new webkitSpeechRecognition();
recognition.lang = 'ko-KR';
recognition.interimResults = false;

recognition.onend = () => {
  // 사용자가 말을 멈춤 → 다음 턴으로 자동 진행
  goToNextLine();
};

recognition.start(); // 사용자 턴 시작 시 호출
```

- 장점: 라이브러리 없음, 브라우저 내장, 구현 10줄
- 단점: Chrome 전용, 인터넷 연결 필요, 결과 텍스트 사용 불가 (트리거 용도로만 씀)
- **MVP 추천**

**옵션 B — @ricky0123/vad-web**

```javascript
import { MicVAD } from "@ricky0123/vad-web";

const vad = await MicVAD.new({
  onSpeechEnd: (audio) => {
    goToNextLine(); // 말이 끝나면 자동 진행
  },
});
vad.start();
```

- 장점: 더 정확한 음성 감지, 크로스브라우저
- 단점: 라이브러리 추가 필요, ONNX 모델 로드 (~1-2초)
- Chrome 이외 브라우저나 정확도가 더 중요할 때 선택

**둘 다 수동 버튼 fallback을 반드시 유지해야 함.**

## 6. 시스템 구조

```
[브라우저]
  ├── 대본 화면 렌더링
  ├── 현재 줄 강조 표시
  ├── AI 턴: <audio> 태그로 TTS 재생
  └── 사용자 턴: VAD 활성화 → 말 끝나면 next()

[백엔드 - FastAPI]
  ├── GET  /api/script          → 대본 JSON 반환
  ├── POST /api/tts             → 텍스트 → TTS mp3 생성 및 반환
  └── GET  /audio/{filename}    → 생성된 mp3 파일 서빙

[TTS]
  └── OpenAI TTS API (tts-1)
```

```

│  ☑ 민준: 오래 기다렸어?             │  ← 완료된 줄 (흐리게)

│  ▶ 민준: 다행이다. 오늘 많이 떨려?  │  ← 현재 AI 재생 중 (강조)
│  □ 지수: 조금. 근데 괜찮아...       │  ← 다음 내 대사 (대기)
│  □ 민준: ...                        │

[ 🎤 듣고 있어요... ]  ← 내 턴일 때 표시
```

## 8. API 명세

### GET /api/script

```json
// 응답
{
  "title": "첫 만남",
  "characters": ["지수", "민준"],
  "lines": [
    { "index": 0, "character": "민준", "text": "오래 기다렸어?" },
    { "index": 1, "character": "지수", "text": "아니, 나도 방금 왔어." }
  ]
}
```

### POST /api/tts

```json
// 요청
{
  "text": "오래 기다렸어?",
  "character": "민준",
  "line_index": 0
}

// 응답
{
  "audio_url": "/audio/line_0_minjun.mp3"
}
```

- 같은 텍스트+캐릭터 조합은 파일이 이미 존재하면 재생성하지 않고 캐시된 URL 반환
- 음성 파일은 서버 `audio/` 디렉토리에 저장

## 9. 구현 순서 (추천)

1. **백엔드**: 대본 JSON 반환 API (`/api/script`)
2. **백엔드**: TTS 생성 + 파일 캐시 API (`/api/tts`)
3. **프론트**: 역할 선택 화면
4. **프론트**: 리허설 화면 렌더링 (대본 표시)
5. **프론트**: AI 턴 — TTS 요청 → `<audio>` 재생 → 재생 완료 시 다음 턴
6. **프론트**: 사용자 턴 — VAD 시작 → 말 끝 감지 → 다음 턴
7. **프론트**: 수동 진행 버튼 (fallback)
8. **프론트**: 완료 화면

## 13. 알려진 제약사항

- Web Speech API는 **Chrome 브라우저 전용**. Safari / Firefox는 수동 버튼으로 진행
- VAD는 말을 인식할 뿐 내용 검사는 하지 않음 (배우가 다른 말을 해도 넘어감)
- TTS 생성은 첫 실행 시 약 1-3초 딜레이 발생 (이후 캐시 사용)
- HTTPS 환경에서만 마이크 접근 가능 (로컬 개발은 `localhost` 허용)
